# 实现主板单独显示测试进度信息

## 问题分析

当前系统存在以下问题：

1. 测试进度信息（当前命令、当前测试项、测试轮次）是全局显示的，所有主板共享同一个进度
2. 不同主板的测试进度可能不同，但当前界面无法区分
3. 用户希望每个主板在表格中单独显示自己的测试进度信息

## 解决方案

### 1. 修改 TestProgress 类

* 添加 `host_progress` 字典，用于存储每个主板的测试进度

* 每个主板的进度包括：当前命令索引、当前测试项、当前轮次、总轮次等信息

### 2. 更新 TestExecutor 类

* 修改测试执行逻辑，为每个主板单独跟踪测试进度

* 在执行测试时，更新对应主板的进度信息

* 修改进度回调机制，传递包含所有主板进度的信息

### 3. 修改 GUI 界面

* 更新测试控制表格，在主板名称列中显示测试进度信息

* 当主板正在测试时，显示格式：`主板名称 (当前命令/总命令 - 当前测试项 - 当前轮次/总轮次)`

* 当主板未测试时，只显示主板名称

### 4. 更新进度更新逻辑

* 修改 `_update_progress` 方法，处理每个主板的进度信息

* 在表格中实时更新每个主板的测试状态和进度

## 实现步骤

1. **修改 test\_executor.py**

   * 更新 TestProgress 类，添加 host\_progress 字段

   * 修改测试执行方法，跟踪每个主板的测试进度

   * 更新进度通知机制

2. **修改 main.py**

   * 更新 `_update_progress` 方法，处理每个主板的进度信息

   * 修改 `_update_host_info` 方法，在表格中显示主板测试进度

   * 更新表格数据结构，支持显示测试进度信息

3. **测试验证**

   * 确保多主板并行测试时，每个主板的进度信息正确显示

   * 验证测试状态更新的实时性

   * 确保界面美观性和信息可读性

## 预期效果

* 每个主板在表格中单独显示自己的测试进度

* 不同主板的测试进度互不影响

* 用户可以一目了然地看到每个主板的当前测试状态

* 界面更加清晰，信息更加直观
